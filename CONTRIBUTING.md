# Contributing to uniapp-mini-program-skill

## 如何贡献

欢迎提交 Issue 和 Pull Request。本项目是一个“踩坑记录型”Skill，任何你在 uni-app 小程序开发中遇到的真实坑、且找到了解决方案，都值得加入。

## 添加新规则的标准

1. 问题必须是**平台相关**的（微信/支付宝/抖音等差异导致）
2. 必须有**可复现的条件**（比如特定的 API 调用顺序、生命周期组合）
3. 必须给出**具体的排查步骤和解决方案**

## 规则文件格式

请在 `rules/` 目录下创建 `.md` 文件，文件名用英文短横线连接，例如：`alipay-requestpayment-toast.md`

文件内格式：

```markdown
# 问题名称
## 现象
## 适用平台
## 原因
## 解决方案
## 相关链接（可选）
```

## 代码风格

无特殊要求，清晰即可。

## 提交前检查

- 问题不是单纯的使用错误，而是平台/框架的坑
- 解决方案经过实际验证
